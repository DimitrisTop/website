01 · The Problem & Concept

The brief was open-ended: use AI to solve a real human problem. Rather than defaulting to a generic chatbot or data interface, we framed the problem around a specific daily friction — deciding what to wear in a way that accounts for environment, schedule, and personal preference simultaneously.

Most styling apps focus only on wardrobe. We defined the real problem as context mismatch: a person's outfit decision is influenced by weather, plans, and mood — none of which static apps account for. That framing drove the entire solution design.

02 · My Contribution: Design & Coordination

My role was not execution-only. I contributed to three distinct areas:

  • Problem Framing: Defined the product concept and the core use case. Established which user inputs (wardrobe, weather, plans) were essential and how they relate to each other in the recommendation logic.
  • User Interaction Flow: Mapped the end-to-end interaction: from user onboarding and wardrobe input through to context-aware recommendation delivery. Identified the decision points and data dependencies at each step.
  • Process Orchestration: Structured the system logic connecting multiple data sources — user profile, wardrobe inventory, live weather, and calendar context — and coordinated how they feed into the AI recommendation layer.

03 · System Logic & Architecture

The technical implementation used Microsoft Azure as the orchestration backbone. The key design decision was modularity: rather than a monolithic backend, each data source (weather API, user input, wardrobe database) was isolated as a separate component. This meant a failure in the weather API would not take down the core recommendation engine.

  • Azure OpenAI (GPT-4): Stylistic analysis and natural-language interaction.
  • Azure Logic Apps: Orchestrating the flow between user input, environmental data, and the recommendation engine.
  • Azure Functions: Processing image metadata for outfit context recognition.

04 · Outcome & Recognition

Awarded the Silver Medal by PwC judges for product strategy and cross-functional collaboration. The evaluation criteria centered on how well the team defined a real business problem, designed a coherent solution, and coordinated across different technical domains under time pressure.

The core lesson: in a hackathon, the teams that win are those who spend the first hour defining the problem clearly — not the ones who start building immediately.
